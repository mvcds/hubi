Domain File(A.K.A Configuration File)

An YAML file containing information about a particular piece of the ubiquitous language.

Not all domain files translate to source files due to the "abstract" configuration e.g. this very one!

Ubiquitous Token(A.K.A Token, Domain Object, Object Value, Entity)

A piece of the ubiquitous language extracted from its domain file

It represents a phrase, expression, term, etc

Name Type Default Description Comment
name string How people refer to the token
(Should be unique across the domain to avoid ambiguity)
description string A short sentence to teach/remind everyone what the token means
attributes array A collection of attributes a token is associated with.
This collection is specially useful for developers and is where translators are expected to work more often.
filePath string The path to its file domain
abstract boolean false A flag to inform translators if they should or not translate the token
comment string An extra message, for developers, about the token
aliases array Alternative names, abbreviations, etc
Other ways to refer to the token itself
Attribute(A.K.A Characteristic, Member)

A token's characteristic.

Programming-wise, it is an object's member.

Name Type Default Description Comment
name string How the attribute is referenced relativelly to its token
type string "string" What kind of token it is
required boolean false A flag to mark the attribute as required
description string A short sentence to teach/remind everyone what that attribute means
comment string An extra message, for developers, about the attribute
