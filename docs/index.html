Domain File

A file containing information about a particular piece of the ubiquitous language which will be transformed into a token

  • name: How people refer to the token (should be unique across the domain to avoid ambiguity)
  • description: A short sentence to teach or remind everyone what that token means
  • attributes: A collection of attributes a token is associated with (this collection is specially useful for developers)
Ubiquitous Token

A piece of the ubiquitous language (phrase, expression, term, etc) extracted from the domain file

  • object: The object held by the token
Attribute

A token's characteristic

  • name: How the attirbute is referenced relativelly to its token
  • type: What kind of token it is
  • required: A flag to mark the attribute as required when needed
  • description: Aa short sentence to teach or remind everyone what that attribute means
Translator

An object which translates the tokens' into relevant source files for developers

  • ubiquitousLanguage: The language the translator will work on
Ubiquitous Language

A common language for everybody

  • tokens: A collection of ubiquitous tokens
